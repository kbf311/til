# Gitのコミット履歴をクリアして新しくやり直す手順

この手順を実行すると、過去の全コミット履歴が削除され、現在のファイル状態を「最初のコミット」とした新しいリポジトリ状態になります。

> ⚠️ **注意:** 過去の履歴が完全に消去され、リモートリポジトリに強制プッシュ（上書き）します。チームで共有しているリポジトリの場合は、事前に必ずメンバーに確認してください。

---

## 実行コマンド

以下のコマンドを順番に実行してください。

```bash
# 1. 履歴を持たない、完全に独立した新しい一時ブランチ（latest_branch）を作成
git checkout --orphan latest_branch

# 2. 現在のディレクトリにあるすべてのファイルをステージング領域に追加
git add -A

# 3. 新しい「最初のコミット」を作成
git commit -m "Initial commit"

# 4. 古い履歴が残っているローカルの main ブランチを削除
git branch -D main

# 5. 現在の一時ブランチ（latest_branch）の名前を「main」に変更
git branch -m main

# 6. リモートリポジトリ（origin）の main ブランチに強制プッシュ（上書き）
git push -f origin main
```
